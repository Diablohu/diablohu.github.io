基础框架

以下是 PRPRACG 页面 body 基础的结构。

主体内容容器#layout

所有页面内容均放入 #layout 容器中。利用 #layout 容器可以方便的调整 z-index 层级等与其他内容的 css 关系。

页面头部#header

页面头部有多种样式，当前支持以下格式，点击相应按钮可切换：


  • 无需指定 #header 的 class

  • #header.mod-banner

  • #header.mod-title

  • #header.mod-circle，代码结构会发生变化，详见圈子页

  • #header.mod-post，代码结构会发生变化，详见帖子详情页

页面主体#main

建议在 #main 容器上设定表示当前页面 class，方便 LESS 和 JS 控制本页面。

页面主体的内容颜色默认为 @base-color-main。

基准宽度控制

在上文中可以看到使用了很多 .layout-wrapper 容器，这一容器可控制基准宽度。如果要使用占用整个页宽同时受页面宽度控制统一管理的元素，如上面提到的页面主体强调内容 .main-banner，则可以考虑使用该容器。

使用边框宽度 border-width CSS 属性控制两侧间距，其子元素可以放心的使用 position: absolute 设置浮动。

基准宽度规则：

  • 默认：宽度 90%，最大宽度 @layout-width-max
    • 对于不支持 media query 的老式浏览器，设有 @screen-t1 的最小宽度。
  • viewport 宽度不大于 @screen-t1 时：宽度 100%，两侧间距 30px
  • viewport 宽度不大于 @screen-mobile 时：两侧间距 24px
  • viewport 宽度不大于 @screen-t5 时：两侧间距 16px

 

网格 (Grid)

网格框架（Grid）现有 2 种模式：侧边宽度固定的正文/侧边布局和各网格宽度自适应的活动布局。

共享规则

  • 容器元素（.row / .row-2col / .row-4div）的宽度为 auto，为保证排版不出现问题，容器元素要求：
    • 不应赋予宽度
    • 不应浮动
  • 如果有上述需求，可考虑为容器元素包裹一层父级元素。

正文/侧边布局

  • 侧边栏固定 300px 宽度，正文栏宽度自适应。
  • 侧边栏放置在左边或右边不受限制。
  • 为保证最佳效果，请确保侧边栏的内容高度不高于正文栏。
  • 当 viewport 宽度不大于 @screen-t2 时，正文与侧边均占用一整行空间。

活动布局

  • 容器元素的宽度空间被 12 等分。
  • 子元素需要同时给出 2 个样式名：.col .col-数字
  • 子元素样式名上的数字（.col-1 / .col-7）表示该子元素占用多少空间，1 为 1/12，2 为 2/12，以此类推。
  • 所有子元素在同一行中顶部平齐。
  • 当 viewport 宽度不大于 @screen-mobiew 时，所有子元素均占用一整行空间。
  • 可在容器元素通过样式名设置子元素间距，默认为 30px。如需设置，添加样式名 .row-gutter-数字，如 .row-gutter-15，数字表示间距像素。目前支持：
    • 10
    • 15
    • 20

活动布局 - N等分

  • 规则继承自默认活动布局。
  • 容器元素式名上的数字（.row-4div）表示被 N 等分，目前支持：
    • 4等分 - .row-4div
    • 6等分 - .row-6div
    • 16等分 - .row-16div
  • 子元素占用多少空间计算的分母为容器元素 N 等分的数字。
  • 当 viewport 宽度不大于 @screen-mobiew 时，子元素排布会自适应，而非默认活动布局的子元素占用一整行空间，如：
    • 4等分时，子元素占用一半行空间
    • 6等分时，子元素占用 1/3 行空间

命名规则

标签语义化，如使用 P 表示正文段落，DL 表示文本列表等。如果无法用标签表达，一律使用 DIV 或 SPAN，视父元素可否包含区块元素而定。

样式名和ID定义要有意义，如 .button 表示按钮，#header 表示头部。

页面专用样式与脚本应以 page- 为开头，如 page-anime-staff / page-user-topics。

针对元素状态的样式应以 state- 为开头，如 state-hover / state-focus。

使元素变形的样式应以 mod- 为开头，如 mod-hide / mod-highlight。
