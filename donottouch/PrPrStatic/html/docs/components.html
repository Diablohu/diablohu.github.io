欢迎来到 PRPRACG 前端开发文档。本页将介绍全站的基础组件，这些标准化元素可任意使用。

基本规则

基础组件共享以下基本规则：

  • 基础组件一般不会设定宽、高尺寸，通常情况下是内容容器，或占满父级容器的宽度。
    • 对于列表排布内容，推荐使用 网格 作为容器，请参阅 全局框架 下的相关内容。
  • 基础组件使用的颜色一般会随页面主题变化。
    • 关于颜色主题的开发规范请参阅 LESS 下的相关内容。

动作

点击访问访问 href 链接地址。读取期间元素添加属性 disabled="progress"，以防止多次触发。

目前支持以下方式：

  • 无指定: 不使用AJAX，直接访问目标URL
  • 添加 .action-main: 不使用AJAX，添加颜色样式
  • 添加 .action-plus: 使用AJAX，调用成功后括号内的数字 +1，并自动添加 .action-plus-on 样式。如果当前为 .action-plus-on，调用成功后阔耐内数字 -1，并将该样式移除
  • 添加 .action-confirm: 使用AJAX，弹出提示框询问是否执行

如果以AJAX方式执行，在调用完成后，互动的元素会触发以下事件函数：

  • actionSuccess( evt, data, textStatus, jqXHR ) 调用成功
  • actionError( evt, jqXHR, textStatus, errorThrown ) 调用发生错误
  • actionComplete( evt, jqXHR, textStatus ) 调用过程完成，无论成功与否
  • actionCancel() 调用过程被取消
标准 标准，主要颜色 喜加一 (0) 需确认

 

保持宽高比的容器

该容器会保持宽高比永远一致。如无特殊设定，其宽度占满父级容器，高度根据长宽比自适应。

第一级子元素的样式名为 .ratio-宽x高。

目前支持的宽高比：3:2、2:3、5:4、4:5、16:9、9:16、16:10、10:16
